Introduction

If you‚Äôve spent years on Windows, you‚Äôve been taught that “gaming on Linux” is either a myth or a headache. But for those of us who grew up tinkering from our young days to the depths of the Arch Wiki, we know the truth. On Arch, you don’t just “run” games… You optimize the entire environment for them.

With an AMD CPU/GPU combo and the right configuration, your Linux machine isn’t just a daily driver, it‚Äôs a high-performance monster machine that you actually own.

1. The Core Arsenal (AMD + Arch)

Unlike Nvidia, AMD drivers are baked directly into the Linux kernel (amdgpu). To get peak performance, you need the Mesa stack, the open-source implementation of OpenGL and Vulkan‚Äîalong with the 32-bit (multilib) libraries. Because many games (and the Steam client itself) are still built on 32-bit architecture, these libraries are mandatory.

The Must-Have Packages

Based on my current system configuration, these are the essential packages that make the magic happen. You can install them using yay:

yay -S steam steam-native-runtime vulkan-radeon lib32-vulkan-radeon \
vkd3d lib32-vkd3d mangohud gamescope gamemode lib32-gamemode \
vulkan-mesa-layers lib32-vulkan-mesa-layers wine winetricks

2. The “Performance Final Boss” ‚Äî CPU Governors

If you want to squeeze every drop of power out of your AMD CPU, you have to manage your CPU Governor. By default, Linux is polite; it uses “Balanced” or “Schedutil” to save energy. But in gaming, that split-second delay while the CPU “ramps up” to handle a heavy load can cause a micro-stutter.

To kill that stutter, we force the CPU into Performance mode. Here is exactly how and why we do it.

1. Verification & Testing

Before changing anything, we check the current state of the system using these commands:

  • gamemoded -t: This runs a test for GameMode. It checks if the daemon is correctly installed and if it has the permissions to change your CPU governor on the fly.
  • cat /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor: This “reads” the system file for every CPU core. If it says powersave or schedutil, your CPU is currently being throttled to save power.

2. The Udev Rule (Hardware Automation)

We use a udev rule to tell the Linux kernel to apply settings the moment it “sees” the hardware.

The Command: sudo nvim /etc/udev/rules.d/99-cpu-governor.rules

The Content: `The Content:

ACTION=="add", SUBSYSTEM=="cpu", KERNEL=="cpu[0-9]*", ATTR{cpufreq/scaling_governor}="performance"

Why?
This ensures that every time a CPU core is detected by the system (at boot or after waking up), the kernel immediately locks it into “performance” mode.

Reloading the rules:
To apply this without rebooting, run:

sudo udevadm control --reload-rules && sudo udevadm trigger

3. The Systemd Service (Persistence)

Sometimes the kernel settings get reset during the boot process by other power management tools. To make sure our settings “stick,” we create a Systemd Service. The Content:

[Unit]
Description=Set CPU Governor to Performance
After=multi-user.target

[Service]
Type=oneshot
ExecStart=/bin/bash -c "echo performance | tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor"

[Install]
WantedBy=multi-user.target

After=multi-user.target: Tells the service to wait until the system is fully booted.
Type=oneshot: The script runs once and exits; it doesn’t stay in memory.
tee: Writes “performance” into the system files for every single CPU core (cpu*).

To activate it:

sudo systemctl enable --now cpu-governor.service

enable: Sets it to start automatically on every boot.
--now: Starts the service immediately.

4. The Manual “Force” commands

If you ever want to change it manually without using a service:

echo "performance" | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor

Why?
This is the “brute force” method. It overrides current settings and forces the hardware to max clock speeds instantly.

Script vs. GameMode

While these scripts are powerful, we also have GameMode installed.
The Scripts: Keep your CPU at 100% speed all the time. Great for absolute stability, but it uses more electricity and generates more heat while you’re just browsing.
GameMode: Only applies these exact settings dynamically when a game is running.

3. The Secret Weapon ‚Äî Proton-GE

While Valve‚Äôs official Proton is the engine that drives Linux gaming, they have to be conservative to ensure stability across millions of Steam Decks. For the real enthusiasts, we have Proton-GE (GloriousEggroll).

Proton-GE is a community build that includes cutting-edge patches, newer media foundations (to fix those broken in-game cutscenes), and specific game-specific fixes that haven’t hit the main Steam branch yet.

Managing Versions with ProtonUp-Qt

Since you have protonup-qt installed on your system, you don’t need to manually move folders or mess with hidden directories. If you don’t have it yet, you can install it using:

yay -S protonup-qt
  1. Launch ProtonUp-Qt: It will detect your Steam installation automatically.
  2. Add Version: Choose the latest GE-Proton (e.g., GE-Proton9-x) and click install.
  3. Restart Steam: This is necessary for Steam to see the new “Compatibility Tool.”

Enabling it for your Games

Once installed, you need to tell Steam to use it. You can do this per game:

  • Right-click your game in Steam > Properties.
  • Go to the Compatibility tab.
  • Check “Force the use of a specific Steam Play compatibility tool.”
  • Select the GE-Proton version you just downloaded.

Why use GE on Arch?

On an Arch system running the Linux-Zen kernel (like yours), Proton-GE shines. It is built to leverage Fsync, which reduces CPU overhead and can significantly improve your 1% low FPS, making the gaming experience much smoother. If you don’t have installed Zen kernel yet, install it by doing:

yay -S linux-zen linux-zen-headers

And then:

sudo grub-mkconfig -o /boot/grub/grub.cfg

Reboot: Restart your computer and select the Zen kernel from the boot menu.

4. Real-time Monitoring & Control ‚Äî MangoHud & Gamescope

We didn’t switch to Arch to have a “black box” experience. We want to see exactly how our hardware is behaving and have the power to fix resolution or scaling issues on the fly.

MangoHud: The Ultimate Overlay

MangoHud is an OpenGL and Vulkan overlay that displays FPS, temperatures, CPU/GPU load, and even frametime graphs. It is far more powerful than the basic Steam FPS counter.

To use it, you simply need to install it:

yay -S mangohud

And add it to your Steam launch options:

mangohud %command%

Why use it?
It allows you to spot thermal throttling or bottlenecking instantly. Because you have vulkan-mesa-layers and lib32-vulkan-mesa-layers installed, it hooks perfectly into almost every game.

Gamescope: The Micro-Compositor

Gamescope is a tool developed by Valve for the Steam Deck, but it is a lifesaver on a desktop running a tiling window manager like Hyprland. It runs your game in a “virtual” desktop. This is incredibly useful if:
A game refuses to stay in fullscreen.
A game has a weird resolution that messes up your Hyprland workspace.
You want to upscale a lower resolution to your native monitor size using FSR.

To use it in Steam:

gamescope -W 1920 -H 1080 -f -- %command%

(Replace 1920 and 1080 with your desired resolution; -f stands for fullscreen.)
